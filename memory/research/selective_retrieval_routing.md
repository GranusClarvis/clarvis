# Selective Retrieval Routing

_Date: 2026-03-28_

Selective retrieval routing is emerging as the practical middle path between classic RAG and brute-force long-context prompting. The older RAG pattern assumes retrieval is always necessary, then optimizes chunking and reranking. Newer work instead asks a better question first: should the model retrieve at all, and if so, what form of context should it fetch? SR-RAG reframes this as knowledge-source selection, treating the model’s own parametric memory as a first-class source. Rather than a binary retrieve/skip decision, it can verbalize internal knowledge, select an external source, and answer in one generation pass, improving accuracy while reducing retrieval frequency. LongRAG complements this by shifting retrieval units from tiny chunks to semantically grouped long units, reducing search burden and preserving context that small chunks destroy. SARA pushes further with a dual representation: exact textual spans for factual precision plus compressed semantic vectors for efficient reranking under tight context budgets. Anthropic’s Contextual Retrieval adds a strong implementation lesson: retrieval quality often improves more from preserving local context during indexing than from adding more retrieval steps. The key insight is that retrieval optimization is becoming a routing problem: dynamically choose between internal memory, contextualized long units, and compressed evidence rather than blindly stuffing top-k chunks into the prompt.
