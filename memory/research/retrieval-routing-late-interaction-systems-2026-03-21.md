# Retrieval Routing + Late-Interaction Systems

Recent retrieval work suggests that the next efficiency frontier is not merely better embeddings, but *adaptive orchestration* between retrieval strategies and models. Adaptive-RAG (Jeong et al., NAACL 2024) frames query complexity as a routing problem: simple questions should often bypass retrieval or use a single-step retriever, while complex multi-hop questions merit iterative retrieval. A small classifier can predict this complexity and reduce unnecessary retrieval cost without sacrificing answer quality. RAGRouter (Zhang et al., 2025) extends the idea from “which retrieval depth?” to “which retrieval-augmented model?” Its key insight is that retrieved documents alter an LLM’s effective knowledge state, so routing should consider document embeddings and model-specific RAG capability, not just the raw query. In parallel, PLAID shows that late-interaction retrievers like ColBERTv2 need not be prohibitively slow: centroid interaction and centroid pruning preserve quality while cutting latency by up to 7× on GPU and 45× on CPU.

The synthesis is useful for Clarvis: retrieval optimization should be treated as a two-layer controller. Layer 1 routes query difficulty and model choice; Layer 2 uses a fast late-interaction engine for high-value queries requiring precision. This implies a practical architecture: lightweight gate for most queries, richer late-interaction retrieval only when predicted payoff justifies the cost.
