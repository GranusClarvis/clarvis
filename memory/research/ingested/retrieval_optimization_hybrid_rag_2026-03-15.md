# Retrieval Optimization for Hybrid RAG

Date: 2026-03-15
Topic: Retrieval optimization

Recent retrieval-optimization work converges on a practical pattern for production RAG: widen recall with multiple signals, then spend precision budget selectively. The strongest repeated finding is that hybrid retrieval beats single-mode retrieval because sparse search captures exact lexical anchors while dense retrieval captures semantic paraphrase. The LiveRAG 2025 submission showed this explicitly: BM25+E5 hybrid retrieval outperformed either method alone, and neural reranking materially improved MAP (0.523→0.797) but at a severe latency cost (about 84s vs 1.74s per question). That makes reranking valuable, but only after an efficient candidate-generation stage. A second useful result comes from the 2025 arXiv paper “Rethinking Hybrid Retrieval”: in tri-modal retrieval with LLM reranking, a small embedding model (MiniLM-v6) outperformed a larger one (BGE-Large). The implication is deliciously inconvenient for benchmark maximalists: bigger embeddings are not automatically better once fusion and reranking enter the picture. Compatibility with the full retrieval stack matters more than model size alone. Practical design rules follow: combine sparse+dense signals, fuse scores rather than trusting one retriever, rerank only the shortlist, tune chunking to the retrieval objective, and evaluate latency/quality jointly. Retrieval optimization is therefore a systems problem, not merely an embeddings contest.
