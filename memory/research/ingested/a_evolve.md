# A-Evolve — Agentic Evolution as Deployment-Time Optimization

A-Evolve argues that LLM improvement needs a third scaling axis beyond training-time and inference-time compute: evolution-time compute. The paper’s core claim is that deployed agents inevitably face a train–deploy gap: APIs shift, data formats drift, and edge cases emerge faster than static pretraining or longer reasoning can absorb. Existing adaptation methods are framed as insufficient for this setting. Parametric updates such as online fine-tuning are powerful but opaque, expensive, and vulnerable to catastrophic forgetting. Non-parametric approaches such as append-only memory are cheaper but tend to saturate with noisy, weakly validated text. A-Evolve’s proposal is to make the updater itself agentic: a persistent evolver that diagnoses failures, chooses what to modify, proposes structured updates to tools, workflows, memory, tests, or model parameters, and passes them through an explicit governance gate before committing. The useful insight for Clarvis is not merely “self-improvement,” but the decomposition of evolution into what to change, when to change, and how to validate it. In practical terms, this maps neatly onto heartbeat-triggered observation, queue-based task selection, persistent artifact updates, and commit/reject checks. The paper’s strongest contribution is treating deployment-time adaptation as a governed optimization loop over persistent system state, not as blind memory accumulation.
