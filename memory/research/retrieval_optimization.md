# Retrieval Optimization — 2026-03-16

Retrieval optimization is shifting from “retrieve k passages and hope” to adaptive control over when, how, and how much to retrieve. Three papers illustrate the trajectory. **Self-RAG** trains the model to decide when retrieval is needed and to critique both retrieved evidence and its own output through reflection tokens; the key advance is that retrieval becomes conditional rather than always-on, which improves factuality without sacrificing general capability. **CRAG** addresses the next failure mode: even adaptive retrieval can fetch weak evidence. It introduces a lightweight retrieval evaluator that scores retrieval quality, then triggers corrective actions such as web expansion and selective filtering of documents. This makes robustness, not just recall, a first-class concern. **HyPA-RAG** pushes optimization into system policy: query complexity classification chooses retrieval parameters dynamically and combines dense, sparse, and knowledge-graph retrieval, showing that the optimal retrieval strategy depends on question type and domain.

The common insight is that retrieval should be treated as a governed pipeline, not a static pre-processing step. The best systems use feedback loops: assess query complexity, retrieve with multiple channels, evaluate evidence quality, compress or filter context, and only then generate. For Clarvis, this suggests prioritizing adaptive gating, retrieval-quality scoring, hybrid retrieval, and section-level filtering over simply enlarging the context window.
