# Clarvis Knowledge Vault

This directory is an **Obsidian-compatible vault** containing Clarvis's three-layer knowledge system: raw sources, compiled wiki pages, and generated outputs.

## Quick Start (Obsidian)

1. Open Obsidian and select **Open folder as vault** → point to this `knowledge/` directory.
2. The `.obsidian/` config is pre-set: graph colors, attachment paths, and core plugins are ready.
3. Install the recommended community plugins (see below).
4. Open the **Graph View** (`Ctrl/Cmd+G`) to explore the knowledge graph.

## Recommended Community Plugins

Install these from Obsidian Settings > Community Plugins > Browse:

| Plugin | Why |
|--------|-----|
| **Dataview** | Query pages by frontmatter (e.g., list all `status: draft` concepts) |
| **Templater** | Use `schema/templates/` for new page creation |
| **Tag Wrangler** | Rename/merge tags across the vault |
| **Graph Analysis** | Betweenness centrality, clustering — find knowledge gaps |
| **Frontmatter Tag Suggest** | Autocomplete tags from the taxonomy in `schema/schema_rules.md` |

Optional but useful:
- **Kanban** — drag wiki tasks between status columns
- **Breadcrumbs** — visualize hierarchy from index pages
- **Calendar** — navigate raw sources and logs by date

## Vault Structure

```
knowledge/
├── .obsidian/          # Vault config (graph colors, core plugins, settings)
├── raw/                # Layer 1 — ingested sources (paper, web, repo, image)
│   ├── paper/          # Academic papers
│   ├── web/            # Web captures
│   ├── repo/           # Repository analysis snapshots
│   └── attachments/    # Binary attachments (images, PDFs)
├── wiki/               # Layer 2 — compiled knowledge pages
│   ├── concepts/       # Ideas, techniques, frameworks
│   ├── projects/       # Software projects and repos
│   ├── people/         # People and organizations
│   ├── syntheses/      # Cross-cutting analysis
│   ├── questions/      # Research questions + answers
│   ├── timelines/      # Chronological sequences
│   ├── procedures/     # How-to guides
│   └── indexes/        # Auto-generated navigation (do not edit)
├── schema/             # Page schemas, templates, rules
│   ├── templates/      # Templater-compatible page templates
│   └── schema_rules.md # Canonical schema reference
├── outputs/            # Layer 3 — generated artifacts (answers, slides)
└── logs/               # Ingest logs, lint reports, source registry
```

## Graph View Conventions

The pre-configured graph view uses color groups to distinguish page types:

| Color | Section | Meaning |
|-------|---------|---------|
| Blue | `wiki/concepts/` | Core knowledge pages |
| Green | `wiki/projects/` | Project/repo pages |
| Orange | `wiki/people/` | People and orgs |
| Red | `wiki/syntheses/` | Cross-cutting analysis |
| Yellow | `wiki/questions/` | Research questions |
| Gray | `raw/` | Source evidence (dimmed) |

**Tips:**
- Use **local graph** on a wiki page to see its immediate neighborhood.
- Orphan nodes (no links) appear isolated — these are candidates for integration.
- Arrow direction shows link direction (enabled by default).

## Creating New Pages

1. Navigate to the appropriate `wiki/` subdirectory.
2. Use Templater (`Alt+N` or command palette) with templates from `schema/templates/`.
3. Fill in the YAML frontmatter — every field in `schema/schema_rules.md` marked "Required" must be present.
4. Add at least one source in `sources:` (path to `raw/` file or external URL).
5. The compile pipeline and lint will validate your page on next run.

## Operator Workflow

### Browse existing knowledge
- Open `wiki/index.md` for the compiled knowledge overview.
- Open `wiki/indexes/tags.md` to browse by topic.
- Open `wiki/indexes/recent.md` for latest updates.
- Use `Ctrl/Cmd+O` (Quick Open) and type a concept name.

### Add a new source
Drop a file into `raw/{type}/` (paper, web, repo) with proper frontmatter, or use the CLI:
```bash
python3 -m clarvis wiki ingest <url-or-file>
```

### Ask a question
```bash
python3 -m clarvis wiki query "How does X work?"
```

### Run maintenance
```bash
python3 -m clarvis wiki lint          # Check for schema violations
python3 -m clarvis wiki rebuild-index  # Regenerate all indexes
```

## Attachment Rules

- Binary files (images, PDFs, data files) go in `raw/attachments/`.
- Reference them from wiki pages as `![desc](../raw/attachments/filename.png)`.
- Do NOT place attachments alongside markdown files in wiki directories.
- The `.obsidian/app.json` config routes new attachments to `raw/attachments/` automatically.

## What NOT to Edit

- `wiki/indexes/` — auto-generated by the compile pipeline. Manual edits will be overwritten.
- `logs/` — written by automated tools. Read-only for operators.
- `.obsidian/` — pre-configured. Adjust only if you know what you're doing.

## Schema Reference

See `schema/schema_rules.md` for the full specification: page types, required frontmatter fields, tag taxonomy, naming conventions, citation rules, and merge/split policy.
