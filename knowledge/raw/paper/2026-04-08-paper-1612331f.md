## Topic 1: Affordance Theory — Gibson

### Core Ideas

**Affordances = Action Possibilities**: An affordance is what the environment offers an agent for action — not an objective property of objects, nor a subjective mental state, but a relational property of the agent-environment system. A chair affords sitting for a human but not for a fish. The same object offers different affordances to different agents depending on body properties, sensorimotor abilities, and cultural knowledge.

**Direct Perception**: Gibson rejected the idea that perception requires internal representations or computation. Instead, organisms directly perceive affordances — they pick up "invariants" in the ambient optic array. Perception is not reconstruction from impoverished stimuli but direct detection of rich environmental structure. This eliminates the representational bottleneck: no need to build an internal model, then reason over it, then act.

**Perceptual Economy**: Affordance-based perception computes only the cues relevant to the agent's goals and action capabilities. Action-agnostic perception would require infinite computational power. By coupling perception to action, the agent filters to what matters — a form of natural attention. This is the opposite of "perceive everything, then decide what's relevant."

### Computational Affordances (Robotics/AI)

Affordance detection in AI has matured significantly. Google's PaLM-SayCan couples LLMs with pre-trained affordance models that estimate which actions are physically possible in the current scene. Affordance-based planning uses LLMs to generate goal states in an affordance domain, then classical planners to execute. The key insight: reasoning at the affordance level (what can I do here?) rather than the feature level (what do I see?) simplifies decision-making by directly linking perception to action selection.

## Topic 2: Ecological Psychology — Agent-Environment Coupling

### Core Ideas

**The Unit of Analysis is the Agent-Environment System**: Ecological psychology rejects the separation of organism and environment. Cognition is not "in the head" — it emerges from the continuous, reciprocal coupling between agent and world. The skilled agent and the landscape of affordances form a single coupled self-organizing dynamical system.

**Perception-Action Loop**: Perception and action are inseparable and interdependent — not sequential stages (perceive → decide → act) but a continuous loop. Action transforms what the agent perceives, and perception guides action. This tight coupling enables real-time adaptation without requiring internal planning or deliberation.

**Sensorimotor Contingencies (SMCs)**: Four distinct levels of sensorimotor coupling:
1. **Sensorimotor Environment** — open-loop motor-induced sensory variations
2. **Sensorimotor Habitat** — closed-loop sensorimotor trajectories
3. **Sensorimotor Coordination** — reliable patterns playing a functional role
4. **Sensorimotor Strategy** — normative organization of coordinations

This hierarchy shows that intelligence doesn't live in a module — it emerges from increasingly structured agent-environment coupling patterns. The agent becomes skilled by developing more stable, reliable coordination patterns.

**Ecological-Enactive Convergence**: Recent work (2020s) shows deep convergence between ecological psychology and enactivism — both reject representationalism, both ground cognition in perception-action coupling, both see the agent-environment system as the proper unit of analysis. The key difference: ecological psychology emphasizes information pickup from the environment; enactivism emphasizes sense-making and value.

## Topic 3: Neural Reuse Theory — Anderson

### Core Ideas

**Massive Redeployment Hypothesis (MRH)**: Brain areas are typically employed to support numerous cognitive functions across diverse task categories. The same neural circuit can participate in language, motor control, social cognition, and mathematical reasoning — simultaneously maintaining all these roles. This is neither strict localization (one area = one function) nor holism (everything does everything), but a middle path.

**Economy Principle**: Given the brain's excessive energy consumption (~20% of metabolic budget for ~2% of body mass), it is more economical to reuse existing circuits than to develop new ones. Evolution and development exploit existing circuits for new purposes through *exaptation* — repurposing without losing original function.

**Reuse Without Local Change**: Circuit acquisition of new functions need not involve unusual circumstances (like injury) or much local structural change. The same circuit can acquire new uses incrementally while maintaining all existing functions. This implies that functional flexibility comes from recombination and context-dependent activation, not from building new hardware.

**Dynamical Motifs as Reusable Computational Primitives**: Recent Nature Neuroscience research (2024) shows that recurrent neural networks solving multiple tasks develop "dynamical motifs" — recurring patterns of neural activity (attractors, decision boundaries, rotations) that are shared across tasks. These motifs enable flexible recombination, promoting modular, compositional computation and rapid transfer learning. The brain reuses not just circuits but computational patterns.

**After Phrenology — A New Vocabulary**: Anderson argues for a multidimensional characterization of brain function, replacing domain-specific labels (language area, motor area) with action-oriented functional descriptions. A brain region's function is described by the full set of tasks it participates in, not by a single label.

## Cross-Topic Synthesis

### Pattern 1: The Relational Turn — Intelligence is Not in the Agent

All three frameworks reject the classical cognitivist view that intelligence lives inside the agent:
- **Gibson**: Affordances are agent-environment relations, not mental representations
- **Ecological psychology**: The coupled system is the unit of analysis
- **Neural reuse**: Function is not localized in circuits — it emerges from context-dependent activation of shared resources across the whole brain

The implication is radical: intelligence is not a property of the processor but of the processor-environment coupling. For an AI agent, this means the quality of the agent-environment interface (tools, memory access patterns, feedback loops) matters as much as the quality of the reasoning engine.

### Pattern 2: Perception-Action Unity Eliminates the Representation Bottleneck

Gibson's direct perception and ecological psychology's perception-action loop both bypass the classical sense → think → act pipeline. Neural reuse provides the mechanism: the same circuits that perceive also act, so there's no translation step. In computational terms:
- Classical: observe → build internal model → reason over model → select action → execute
- Ecological: detect affordances → act (single step)

This maps directly to the difference between a heavyweight planning pipeline and a reactive, affordance-based one. The affordance approach is faster, cheaper, and more robust because it eliminates the representational bottleneck.

### Pattern 3: Reuse as an Organizational Principle

Neural reuse shows that the brain achieves its extraordinary versatility by reusing circuits, not by building new ones. This same principle appears at every level:
- **Gibson**: the same environmental structure affords different actions to different agents — reuse of information
- **Ecological psychology**: the same perception-action patterns apply across contexts — reuse of skills
- **Neural reuse**: the same circuits serve many functions — reuse of computational substrate

For an AI system, this translates to: **build fewer, more general components and reuse them across many tasks**, rather than building specialized modules for each function. This echoes multi-task learning's finding that shared representations prevent overfitting and improve data efficiency.

### Pattern 4: Context-Dependent Activation

In neural reuse, the same circuit produces different outputs depending on which other circuits it's co-activated with. This is parallel to how affordances are context-dependent (the same object affords different actions depending on the agent's current goals and capabilities). The environment provides the context that selects which function a circuit performs — function is determined by the full system state, not by local properties alone.

## Implementation Ideas for Clarvis

### 1. Affordance-Based Task Selection

Replace the current salience-only attention scoring with an **affordance model**: for each candidate task, compute not just "how important is this?" but "what can I actually do with this right now?" Score tasks by the intersection of:
- **Agent capabilities**: which tools/skills/knowledge does Clarvis currently have?
- **Environment state**: which resources are available (brain connectivity, time budget, recent experience)?
- **Action coupling**: how directly can this task connect perception (brain search results) to action (code edit, memory write)?

Tasks where all three align have high "affordance" — they're the ecological equivalent of a ripe fruit within arm's reach. This moves from abstract priority scoring to grounded "can I act on this effectively?" assessment.

**Where to implement**: `attention.py` — add `compute_affordance()` alongside existing `compute_salience()`. Wire into `heartbeat_preflight.py` task ranking so tasks are ranked by `affordance × salience` rather than salience alone.

### 2. Neural Reuse for Brain Collections — Shared Computational Motifs

Currently, Clarvis brain collections are mostly isolated silos with weak cross-collection links (semantic_cross_collection = 0.477). Neural reuse theory suggests the fix isn't more bridge memories (SEMANTIC_BRIDGE task) but **shared computational patterns that operate across collections**.

Implement "reusable motifs" — common operations (pattern matching, analogy detection, causal reasoning) that can be applied to ANY collection, not collection-specific functions. For example:
- A `find_analogies(concept, collection_a, collection_b)` function that reuses the same comparison algorithm across any pair
- "Context-dependent activation" where the same memory appears in different functional roles depending on which task is active — the same memory about "prediction error" functions differently when the active task is about attention vs. when it's about learning

**Where to implement**: `brain.py` — add `reusable_motif()` functions that operate collection-agnostically. The SEMANTIC_BRIDGE task should shift from creating static bridge memories to creating dynamic cross-collection query patterns.

### 3. Perception-Action Tightening — Eliminate the Representation Bottleneck

Current heartbeat pipeline: preflight (perceive) → Claude Code (think) → postflight (record). This is the classical sequential pipeline that ecological psychology critiques. Tighten the loop:
- Make brain search results directly suggest actions (not just provide context)
- Make postflight learning feed immediately back into the next preflight's perception
- Reduce the gap between "what I know" and "what I do" — brain search should return not just memories but **action suggestions** derived from prior episodes where similar situations led to successful outcomes

This is the perception-action loop: let past action outcomes directly shape current perception, which directly guides current action, without an intermediate "deliberation" step.
