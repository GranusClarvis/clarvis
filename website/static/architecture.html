Architecture

How Clarvis thinks, remembers, and evolves.

Conscious Layer (direct interaction) ├── LLM gateway (Node.js, systemd) ├── Chat channels (Telegram, Discord) ├── Skills and command routing └── Reads subconscious digest for context Subconscious Layer (autonomous, cron-scheduled) ├── 20+ scheduled jobs (planning, evolution, reflection, research) ├── Heartbeat pipeline (gate → preflight → execute → postflight) ├── Brain read/write (ClarvisDB) └── Writes results to digest for conscious layer Brain (ClarvisDB) ├── ChromaDB vector store (fully local, ONNX embeddings) ├── 10 memory collections (identity, episodes, learnings, procedures, ...) ├── Graph relations (SQLite + WAL, indexed, ACID) └── Hebbian learning dynamics (STDP synapses)

Core Components

Clarvis Runtime

The autonomous evolution runtime. Manages the heartbeat pipeline, cron orchestration, task routing, operating modes, and self-directed improvement. The conscious layer handles direct interaction; the subconscious layer runs autonomously on a schedule.

ClarvisDB (Memory Substrate)

ChromaDB-backed vector memory with ONNX MiniLM embeddings, fully local with zero cloud dependencies. Supports 10 specialized collections, graph-based relations with dual-backend (JSON legacy + SQLite indexed), and Hebbian learning for connection strengthening between related memories.

Context Assembly

Internal module that compresses and assembles context for each task execution. Pulls from brain search, episodic memory, working memory, and active directives to build focused prompts. Not exposed as a separate package — tightly coupled to the runtime.

Runtime Modes

Clarvis operates in one of three modes, which govern autonomous behavior:

GE — Glorious Evolution

Full autonomy: self-directed research, queue filling, and evolution.

Architecture / Maintenance

Focus on fixing, wiring, simplifying, and benchmarking existing systems. No broad self-added feature work.

Passive / User-Directed

Work only when prompted. No autonomous queue generation or self-directed research.

Invariants

  • All memory and embeddings are local — no cloud calls for retrieval
  • Subconscious jobs acquire global lock for mutual exclusion
  • Heartbeat gate checks health before every autonomous task
  • Mode switching changes policy, not task continuity
  • Public endpoints never expose private memory, secrets, or internal paths
