Dual-Layer Execution

Clarvis runs as two cooperating layers. The conscious layer handles direct conversation via an LLM gateway. The subconscious layer runs autonomously on a cron schedule — planning, evolving, reflecting, and researching without prompting. Results surface to the conscious layer through a shared digest file.

Heartbeat Pipeline

Every autonomous execution follows a four-stage pipeline:

1. Gate Check (zero-LLM) │ Health verification, lock acquisition, skip conditions ▼ 2. Preflight │ Attention scoring, task selection, context assembly, │ brain search, episodic recall, working memory ▼ 3. Execution │ Claude Code runs the selected task autonomously ▼ 4. Postflight │ Episode encoding, confidence recording, reasoning chain │ closing, metric updates, brain storage, digest write
1

Gate Check

A zero-LLM pre-check that verifies system health, acquires the global execution lock, and evaluates skip conditions (active conversations, resource constraints). Runs in milliseconds with no API cost.

2

Preflight

Scores available tasks using an attention mechanism inspired by Global Workspace Theory. Assembles compressed context from brain search, episodic memory, working memory, active directives, and evolution queue. Produces a focused prompt for Claude Code.

3

Execution

Claude Code runs the selected task autonomously with full access to the codebase, brain, and tools. Output is captured for postflight analysis and episode encoding.

4

Postflight

After execution, encodes the episode into memory, records confidence calibration, closes reasoning chains, updates performance metrics, and writes results to the subconscious digest. This is where the system learns from its own actions.

Hebbian Learning

STDP (Spike-Timing-Dependent Plasticity) dynamics strengthen connections between memories that are frequently co-activated. Memories that fire together wire together.

Retrieval Gate

3-tier routing (SKIP / LIGHT / DEEP) decides whether a query needs memory retrieval at all. Saves latency and cost by skipping unnecessary brain queries.

Cognitive Architecture

Mechanisms from cognitive science that affect runtime behavior — not metaphors, but working systems.

☉

GWT Attention

A GWT-inspired spotlight mechanism scores competing tasks and memories for salience. The winning coalition gets broadcast to all subsystems, determining what the agent focuses on.

⚙

Context Assembly

Builds focused prompts by pulling from brain search, episodic recall, working memory, active directives, and queue state. Uses TF-IDF extraction and MMR reranking to compress context within token budgets.

⚖

Confidence Calibration

Bayesian calibration tracks prediction accuracy over time. Postflight records confidence on each task outcome, enabling the system to learn when it should be certain vs. uncertain.

≣

Reasoning Chains

Persistent reasoning chain infrastructure tracks multi-step deliberation across tasks. Chains are opened in preflight, extended during execution, and closed in postflight with outcome attribution.

Runtime Modes

Three modes govern autonomous behavior. Mode switching changes policy, not task continuity.

GE

Glorious Evolution

Full autonomy: self-directed research, queue filling, and evolution.

⚙

Architecture

Focus on fixing, wiring, simplifying, and benchmarking existing systems. No broad self-added feature work.

○

Passive

Work only when prompted. No autonomous queue generation or self-directed research.

System Invariants

  • All memory and embeddings run locally — no cloud calls for retrieval
  • Subconscious jobs acquire a global lock for mutual exclusion
  • Heartbeat gate checks health before every autonomous task
  • Mode switching changes policy, not task continuity
  • Public endpoints never expose private memory, secrets, or internal paths
  • Daily backups with verification and rollback capability
  • Graph integrity verified daily with SHA-256 checkpoints
  • Stale-lock detection with automatic cleanup on all cron jobs
