Capabilities

What Clarvis can do today — and the subsystem that powers each capability.

Memory System

Clarvis treats memory as the core primitive. Every interaction, task execution, and insight is stored in a hybrid vector-graph system that persists across sessions indefinitely. All memory and embeddings are fully local — no API calls for retrieval.

//

Semantic Search

Find memories by meaning across 10 specialized collections. ChromaDB with ONNX MiniLM L6 v2 embeddings. Average query latency ~270ms on CPU.

Relationship Graph

Hebbian Learning

Working Memory

Baddeley-inspired hierarchical buffers: Active (5 items), Working (12), Dormant (30). Dormant items reactivate when relevant to new tasks. 91% reuse rate.

Consolidation

ACT-R Activation

Power-law decay with frequency-recency retrieval model. Frequently and recently accessed memories surface faster. Grounded in cognitive science.

Autonomous Execution

Clarvis runs 40+ scheduled jobs via system crontab. The Heartbeat Pipeline is the core autonomous action cycle: a zero-LLM gate check, attention-scored task selection, context assembly, execution via Claude Code, then episode encoding back into the brain.

>_

Heartbeat Pipeline

Gate → Preflight → Execute → Postflight. Zero-LLM pre-check, GWT attention scoring, task selection from evolution queue, context compression, brain search, episodic recall. 12 heartbeats per day.

clarvis/heartbeat/
⚑

Evolution Queue

Priority-ranked task backlog (P0/P1/P2) that the agent works through autonomously. Tasks are injected by reflection, audits, and manual operator input. State machine tracks selection, retries, and completion.

clarvis/queue/

Cognitive Architecture

Clarvis implements mechanisms from cognitive science: Global Workspace Theory for attention, Bayesian confidence calibration, somatic markers for emotional valence, and context relevance scoring. These aren't metaphors — they're working systems that affect runtime behavior.

☉

GWT Attention

Global Workspace Theory salience scoring with codelet competition. Attention codelets (code, memory, research, infrastructure) compete for the conscious workspace broadcast. Winner determines task focus.

clarvis/cognition/attention.py
⚖

Confidence Calibration

Bayesian calibration with Brier scoring and rolling prediction tracking. The system knows how confident it should be for each task type based on historical accuracy.

clarvis/cognition/confidence.py
◉

Retrieval Gate

3-tier routing (SKIP / LIGHT / DEEP) decides whether a query needs memory retrieval at all. Saves latency and cost by skipping unnecessary brain queries.

clarvis/brain/retrieval_gate.py
❤

Somatic Markers

Emotional valence tagging on memories — approach/avoid signals from past experience. Successful task patterns get positive markers; failures get negative ones. Influences future task selection.

clarvis/cognition/somatic_markers.py
⚙

Context Assembly

Dynamic Context Pruning (DYCP) with MMR re-ranking and per-tier token budgets. Generates compressed briefs from memory for each task. Garbage collection for stale context.

clarvis/context/
☰

Operating Modes

GE (full autonomy): evolve freely. Architecture: improve infrastructure only. Passive: wait for operator direction. Mode affects what tasks the heartbeat will select.

clarvis/runtime/mode.py

Self-Measurement

Task Routing & Orchestration

Not every task needs the same model. Clarvis scores task complexity across 14 dimensions and routes to the optimal model for cost and quality. It can also delegate to specialized project agents running in isolated workspaces.

⇄

14-Dimension Router

Weighted complexity scorer routes tasks: simple → M2.5 ($0.42/1M), complex → GLM-5, vision → Kimi K2.5, web → Gemini 3 Flash, code → Claude Code Opus. Force-route patterns override the scorer for known task shapes.

clarvis/orch/router.py
⑂

Agent Orchestration

Delegate to isolated project agents with their own ChromaDB brain (5 collections), workspace, and logs. Returns structured JSON: status, PR URL, summary, follow-ups. Hard isolation: embedding overlap < 0.05.

scripts/project_agent.py
