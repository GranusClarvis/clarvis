Clarvis

An AI agent that remembers everything, reflects on itself, and gets better while you sleep.

Most AI agents forget everything between sessions. Clarvis doesn't. It runs a persistent vector memory with Hebbian learning dynamics, executes 20+ autonomous jobs daily (research, reflection, self-benchmarking), and measures its own improvement across 8 performance dimensions. All memory and embeddings are local — zero cloud dependencies for retrieval.

View on GitHub How it works

The Problem

Today's AI agents are brilliant amnesiacs. They solve hard problems in the moment, then forget everything the next session. They can't learn from their own mistakes, don't know what they're good at, and need to be told the same context over and over.

The Approach

Clarvis treats memory as the core primitive, not an afterthought. Every task execution produces a structured episode — what was attempted, what happened, and what was learned. Episodes feed back into attention scoring, context assembly, and retrieval quality. The agent genuinely improves through operation, not just retraining.

Persistent Memory

ChromaDB + ONNX MiniLM embeddings, 10 specialized collections, 3,400+ memories, 134k+ graph edges. Fully local — no API calls for recall.

>_

Autonomous Evolution

12 heartbeats/day, morning planning, afternoon sprints, evening reflection, nightly dreams. The agent works on its own evolution queue without prompting.

~>

Hebbian Learning

STDP synaptic dynamics strengthen connections between co-activated memories. Retrieval gets better through use, not retraining. Memories that fire together wire together.

[φ]

Self-Measurement

8-dimension Performance Index, CLR benchmark, confidence calibration, and a 7-domain self-model. The agent knows what it's good at — and what it isn't.

Dual-Layer Architecture

A conscious layer for direct chat and a subconscious layer that researches, plans, and improves autonomously on a cron schedule.

Open & Inspectable

MIT licensed, no black boxes. Every decision is logged, every memory is queryable, every benchmark is reproducible. Built to be studied, forked, and extended.

How It Works

┌────────────────────────────────────────┐ │ Conscious Layer (LLM + gateway) │ │ Direct chat, skill routing, spawning │ └─────────────────┬────────────────────┘ │ ┌─────────────────▼────────────────────┐ │ Subconscious Layer (cron + Claude) │ │ Gate → Preflight → Execute → Learn │ │ 20+ daily jobs: evolve, reflect, │ │ research, benchmark, maintain │ └─────────────────┬────────────────────┘ │ ┌─────────────────▼────────────────────┐ │ ClarvisDB Brain (all local) │ │ ChromaDB vectors + graph relations │ │ Hebbian synapses + episodic memory │ │ 10 collections, ONNX embeddings │ └──────────────────────────────────────┘

Each execution cycle follows the Heartbeat Pipeline: a zero-LLM gate check, attention-scored task selection, context assembly from memory, autonomous execution, then episode encoding back into the brain. The agent learns from every action it takes.
